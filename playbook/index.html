Brand principles

Four words that govern every design decision — copy, layout, interaction, and illustration.

⬛
Direct
Say what you mean. No hedging, no softening, no filler sentences. If it can be cut, cut it.
○
Clear
Any operator should understand any screen without training. Clarity beats cleverness every time.
◎
Human
Finance is a tool, not a destination. Language and visuals stay grounded and approachable.
▲
Precise
Numbers are the product. Every metric, timestamp, and identifier is typeset with monospace precision.
Voice & tone rules
Do
  • Short, declarative sentences — one idea per line
  • Active voice — "Loan approved" not "Your loan has been approved"
  • Specific numbers — "14 days" not "about two weeks"
  • Monospace for all data values, IDs, amounts
  • ALL CAPS DM Mono for section labels only
Don't
  • Marketing hedges — "seamless", "world-class", "powerful"
  • Explain the obvious — label says it, tooltip says it again
  • Mix serif and sans in the same paragraph
  • Use Newsreader below 24px or above weight 500
  • Use secondary palette (sapphire/peridot/topaz) as primary accent

Usage rules
Do
  • One badge per table cell maximum
  • Use dot only for live / real-time states
  • Match variant to semantic meaning — success = positive outcome, danger = problem
  • Keep label one or two words maximum
Don't
  • Use secondary colours (sapphire/peridot/topaz) for standard status states
  • Override the font — DM Mono uppercase is intentional
  • Stack multiple badges on the same row element
  • Use as a button or interactive element

Using in Code

How to install, import, and consume the design system in any Able Group frontend repo.

Installation
1
Configure GitHub Packages authentication
Add a .npmrc to your repo root:
# .npmrc
@ablegroup:registry=https://npm.pkg.github.com
//npm.pkg.github.com/:_authToken=${GITHUB_TOKEN}
2
Install the package
npm install @ablegroup/design-system
3
Import the token CSS once in your app root
// Next.js: pages/_app.js or app/layout.js
// Vite:    src/main.jsx
// Plain:   <link rel="stylesheet" href="...">

import '@ablegroup/design-system/tokens.css'
4
Import components
import { Badge, Button, Card } from '@ablegroup/design-system'

export default function MyPage() {
  return (
    <Card>
      <Badge variant="success" dot>Active</Badge>
      <Button variant="primary">Approve loan</Button>
    </Card>
  )
}
5
Use tokens directly in your own CSS
/* Use CSS custom properties from the token layer */
.my-panel {
  background: var(--able-surface);
  border: 1px solid var(--able-border);
  color: var(--able-text);
  font-family: var(--able-font-body);
  box-shadow: var(--able-shadow-md);
}

/* Dark mode adapts automatically */
[data-theme="dark"] .my-panel {
  /* no changes needed — tokens handle it */
}
Dark mode

Set data-theme="dark" on <html>. All tokens adapt. Do not re-declare token values in your own dark mode — extend the system instead.

// Detect + persist user preference
const saved = localStorage.getItem('theme')
const preferred = window.matchMedia('(prefers-color-scheme: dark)').matches ? 'dark' : 'light'
document.documentElement.setAttribute('data-theme', saved ?? preferred)
Contributing
Adding a token
  • Edit src/tokens/colors.css or the relevant token file
  • Add both light and dark mode values
  • Prefix all variables with --able-
  • Bump the minor version and update playbook
Adding a component
  • New folder under src/components/
  • Component + CSS module + Story file
  • Consume tokens only — no hardcoded hex
  • Export from src/index.js and update playbook
